## Warum nicht jetzt?

ADHS-Hyperfokus auf neue Features ist verlockend. Aber:

- Vier bestehende Apps müssen erst stabil laufen
- clar·log braucht anwaltliche Freigabe + DSFA
- Drei Apps haben noch keine rechtlichen Links
- DPAs sind noch in Beschaffung
- Es gibt noch keine zahlenden Kunden
- App-Store-Launch steht noch aus

**Erst das bestehende System fertig, dann erweitern.**

## Wie diese Konzepte aktuell halten

Wenn dir während der Arbeit an den bestehenden Apps Ideen kommen, die zu clar·task oder clar·community passen:
- Hier ergänzen, nicht in den App-Code einbauen
- Datum + kurze Notiz
- So sammeln sich Ideen, ohne den Fokus zu zerstören
